Docker SAE203 - Site Web de Vidéo à la Demande

Ce projet implémente un site web de vidéo à la demande, hébergé dans un conteneur Docker. Il utilise Apache pour servir les fichiers du site, avec des pages d'inscription, de connexion et de profil utilisateur.

Prérequis

Avant de commencer, assurez-vous que vous avez installé les éléments suivants sur votre machine :

  • Docker : Assurez-vous que Docker est installé et fonctionne correctement. Télécharger Docker
  • Git : Vous aurez besoin de Git pour cloner ce dépôt. Télécharger Git

Installation

  1. Cloner le dépôt : 
    git clone https://github.com/NotSayk/docker-sae203.git
cd docker-sae203
  2. Construire l'image Docker : 
    docker build -t mon_site_web .
  3. Lancer le conteneur Docker : 
    docker run -d -p 8080:80 mon_site_web
  4. Accéder au site : http://localhost:8080

Fonctionnalités

Pages disponibles

  • Page d'accueil (index.html) : La page principale du site
  • Page de connexion (login.html) : Permet aux utilisateurs de se connecter
  • Page d'inscription (signup.html) : Permet de créer un compte
  • Page de profil (profile.html) : Affiche les informations du profil

Scripts JavaScript

  • auth.js : Gère la logique de connexion et d'inscription
  • script.js : Fonctions générales utilisées sur plusieurs pages
  • session.js : Gère la session utilisateur

Fonctionnalités détaillées

Page d'accueil (index.html)

  • Affichage d'une grille de miniatures vidéo avec leurs titres
  • Fonctionnalité de recherche en temps réel
  • Prévisualisation des vidéos au survol
  • Compteur de vues et de likes
  • Indication de la durée des vidéos

Page vidéo (video.html)

  • Lecture avec contrôles standard
  • Affichage des statistiques (vues, likes)
  • Système de likes pour utilisateurs connectés
  • Section de commentaires
  • Boutons de partage et téléchargement
  • Comptage automatique des vues

Système d'authentification

Page d'inscription (register.html)

  • Création de compte avec nom d'utilisateur et mot de passe
  • Vérification de l'unicité du nom d'utilisateur
  • Stockage sécurisé dans le localStorage

Page de connexion (login.html)

  • Authentification avec identifiants
  • Gestion de session via sessionStorage
  • Redirection après connexion

Page de profil (account.html)

  • Affichage et modification de la photo de profil
  • Statistiques personnelles
  • Option de déconnexion
  • Interface personnalisée
